## Overview

Redesigning Tesla's charging experience with AI to reduce driver anxiety, optimize routing, and increase network throughput.

<Callout type="insight">
This case study was produced for Tesla's(China) PM final-round interview. I treated it as a real product initiative — full research, strategy, metrics framework, hi-fi wireframes, and system architecture. Fully self-produced.
</Callout>

<Divider />

## Problem Statement

China's EV charging network has won the coverage battle (12.8M devices, 98% expressway coverage) but lost the experience one. Drivers face a simple, repeating problem: too much information, too much uncertainty, too many decisions, made alone.
User research surfaced three uncertainties present in every charging session:

- **Availability** — When I get there, can I actually charge?
- **Time** — Should I wait? How long?
- **Value** — Is this the best choice?

<Divider />

<Divider />

## The Insight

The real problem isn't missing information. It's that drivers are forced to repeatedly make complex decisions under high uncertainty without help.
The product opportunity is reframing charging from an information lookup tool into a decision system that has an opinion — explainable, accountable, and always dismissible.

## Process

<TwoColumn>
<Column>

### Phase 1 — Research

Mapped the current charging journey end-to-end. Analyzed competitive approaches from ChargePoint, Rivian, and Google Maps EV routing.

Key findings:
- Drivers make charging decisions based on anxiety, not optimization
- Real-time charger availability is the single most requested feature in EV forums
- Existing in-app routing underweights time-of-day demand patterns

</Column>
<Column>

### Phase 2 — Strategy

Defined the AI product vision: **predictive charging that eliminates decision burden**.

Opportunity sizing based on fleet scale (~5M vehicles), average charging sessions per week, and estimated session abandonment rates. Prioritized three intervention points using an impact/effort matrix.

</Column>

### Phase 3 — Design

Created Tesla App-style wireframes for three core flows:
1. **Smart Route Planner** — AI-optimized charging stops based on real-time demand, battery state, and driver preferences
2. **Live Station Dashboard** — queue prediction, estimated wait, and alternative station suggestions
3. **Proactive Notifications** — push alerts when a nearby station is about to become available

### Phase 4 — Architecture

Designed the system architecture connecting vehicle telemetry, charger network data, and the AI prediction layer.

Key components:
- **Fleet Telemetry Ingestion** — real-time battery state, location, and destination from active vehicles
- **Demand Prediction Model** — time-series forecasting of station utilization using historical patterns + live fleet data
- **Routing Optimization Engine** — multi-objective optimization balancing driver convenience, charge speed, and network load

## Key Decisions

<Callout>
**Decision 1: Predictive routing vs. reactive status updates**

Options considered: (A) Show real-time charger availability only, (B) Predict availability 15–30 min ahead and pre-route drivers.

Chose B — reactive status helps at the moment of arrival, but predictive routing solves the problem upstream. Drivers decide their route before they need a charger, not after. This also distributes load across the network proactively.
</Callout>

<Callout>
**Decision 2: In-app experience vs. in-car-only**

Options considered: (A) Build entirely within the in-car display, (B) Build within the Tesla mobile app, (C) Both with app as primary.

Chose C with app as primary — trip planning happens before getting in the car. The mobile app is where drivers make route decisions. In-car display mirrors and adapts the plan in real time. This avoids the constraint of in-car UI iteration cycles.
</Callout>

<Callout>
**Decision 3: AI confidence transparency**

Options considered: (A) Show predictions as definitive ("3 min wait"), (B) Show predictions with confidence ranges ("3–8 min wait, 85% confidence").

Chose B — trust is critical for adoption. Showing confidence ranges sets appropriate expectations and reduces frustration when predictions aren't exact. Over time, as model accuracy improves, ranges tighten and user trust compounds.
</Callout>

## Metrics Framework

Defined success metrics across three layers — user experience, network efficiency, and AI performance:

<Stats items={[
  { value: "↑ Target", label: "Charger Utilization Rate" },
  { value: "↓ Target", label: "Session Abandonment Rate" },
  { value: "≥ 85%", label: "AI Prediction Accuracy" },
  { value: "↓ Target", label: "Avg. Driver Wait Time" }

**User Experience**
- Reduction in average wait time at Superchargers
- Increase in route plan adoption rate (drivers following AI-suggested routes)
- Net Promoter Score improvement for charging experience

**Network Efficiency**
- Charger utilization rate across the network (target: more even distribution)
- Session abandonment rate (drivers leaving a station without charging)
- Peak-to-average load ratio at high-traffic stations

**AI Performance**
- Prediction accuracy for station availability (target: ≥85% within 5-min window)
- Route optimization acceptance rate
- Model latency (predictions served within 200ms)

## Reflection

With more time and access to Tesla's internal data, I would:

- Run a simulation against historical fleet telemetry to validate demand prediction accuracy before any UI work
- Conduct rider interviews focused specifically on charging decision-making under uncertainty
- Explore dynamic pricing as a demand-shaping lever (controversial but high-impact)
- Design the escalation path for edge cases — what happens when the AI is wrong and a driver arrives at a full station
